# 深度分析画像 — 结构模板

这是报告的固定骨架，改编自机构内部真实尽调画像样本。每次生成都遵循这个结构；章节标题文字可以按语境微调（比如把"技术底盘"换成"产品与商业模式"），但章节顺序和覆盖范围不变。

## 封面
- 机构/团队标签（小字，顶部）
- 公司名（大字，居中，可用英文/项目代号）
- 报告类型："深度分析画像" 或用户指定的名称
- 一行领域标签（例如 "AI for Science · Bio LLM 基座公司"）——用一句话点出赛道+定位，不要泛泛而谈
- "深度画像" 副标题
- 日期
- 保密声明（机构名+"内部资料"）
- 免责声明一行："本材料仅供XX内部参阅，不构成投资建议"

## 目录
列出五个正文章节 + 附录，附近似页码（Word 会自动更新为真实页码，不用手算精确）。

## 一、公司概览与定位
### 1.1 一句话定位
给公司一个精炼的类比定位，最好是"对比同赛道的几代范式/几个标杆"来说明公司在演进链条上的位置（样本用了"第一代 AlphaFold → 第二代 Biomni → 第三代 Bio Foundation Model"这种三段式类比）。如果找不到自然的代际类比，退而求其次，用"是X，不是Y"的排除法给出定位。这一节必须让读者在10秒内明白"这家公司到底在做什么、和谁不一样"。

### 1.2 公司基本面
表格，两列（维度/信息），至少覆盖：
公司全称 / 方向（赛道细分） / 阶段 / 里程碑目标（M1/M2/M3 或等价的阶段目标）/ 团队规模 / 融资阶段 / 对标公司（国内外对应打法的公司）。
未披露的字段照实写"未披露"，可以加"（推测……）"给出合理推断但要标明是推断。

### 1.3 最新动态
最近一次材料更新里出现的新信息（新论文/新客户/新融资/新产品）——如果材料本身没有明确的"最新动态"，可以用最近的公开新闻或已披露的里程碑代替，但要基于真实可查的信息，不要编造时间点。

## 二、核心团队
表格：人物 / 角色 / 核心资历。核心资历要具体到机构名+年限+职级+可验证的成果（论文数、专利数、上一家公司估值、代表案例），不要写"资深"、"经验丰富"这类空话。这是本报告里最需要靠网络公开信息补全和交叉验证的部分——用search核实教育背景、任职经历、过往项目战绩，标注哪些是BP自称、哪些是公开可查证实的。

结尾必须有一段**团队判断**：职能配置是否有盲区（技术/学术资源/组织管理/融资战略这几个角色是否都有人覆盖）、团队密度在同赛道里处于什么水位、以及至少一条需要关注的风险（关联公司的IP/资源边界、核心成员的时间投入度、股权结构等）。

## 三、技术底盘 / 产品与商业模式（根据公司类型二选一，见 SKILL.md 的 "Adapting Section 3"）

### 技术驱动型公司版本
- **总览**：如果技术分多层（比如"基础设施层→模型层→应用层"），先给一段总览说明各层关系和整体闭环逻辑。
- **每一层技术**一个小节：核心创新（技术要点，不要只是复述摘要，要提炼出"这篇论文/这个技术解决了什么别人没解决的问题"）；关键数据（具体benchmark数字+对比对象）；投资含义或差异化定位（这项技术在商业化/护城河/竞对对比里意味着什么）。
- **论文/专利/成果矩阵全景**：一张大表，列出全部核心成果，字段：名称 / 发表渠道或状态 / 所属技术层 / 一句话亮点。这张表的价值在于让读者一眼看到"产出密度"，即使正文只深挖了其中几项。
- 结尾**投资含义综合判断**，至少覆盖：闭环完整性（是否自主可控、依赖外部到什么程度）、方法论统一性（是否有一以贯之的技术路线，还是东拼西凑）、TAM扩展空间、核心不确定性（哪怕技术再强，也要指出一条尚未验证的关键假设）。

### 产品/商业驱动型公司版本
- **产品架构**：核心产品/服务是什么，解决什么具体问题，交付形态。
- **核心指标与traction**：留存、GMV/ARR、用户/客户数、增长曲线——凡是BP给出的数字都要标注时间点和口径；凡是行业常见但BP没给的关键指标（比如SaaS的NRR、电商的复购率）要明确写"未披露，需在尽调中索取"。
- **商业模式与单位经济**：怎么赚钱、毛利结构、获客成本与LTV（如有）。
- 结尾**商业化含义综合判断**：单位经济是否已经跑通、规模化路径是否清晰、核心不确定性是什么。

## 四、竞争格局与差异化定位
### 4.1 主要竞品对比
表格：玩家 / 打法路径 / 关键数据（融资、估值、产品进展等，来自公开信息） / 与目标公司的差异化。竞品要覆盖"最像的直接对手"和"用不同方法解决同一问题的对手"两类，不要只列一种。

### 4.2 关键差异化事件
挑1-3个最能说明差异化的具体技术/产品/事件，逐条展开对比逻辑（"对手做的是X，目标公司做的是Y，本质区别在于Z"），比空泛地说"我们更强"更有说服力。

### 4.3 竞争护城河评估
表格：护城河维度（方法论/技术原创性、团队密度、数据飞轮、生态/开源、客户关系等）/ 强度（高/中/低） / 评估依据。刻意包含至少一项"低"或"中"的维度——护城河评估如果全是"高"就是没有做真实评估。

结尾**竞争格局核心判断**：最强的1-2个护城河维度是什么，最弱的1-2个是什么，以及什么外部事件（大厂入场、政策变化、替代技术出现）会最快侵蚀当前的领先窗口。

## 五、投资判断
### 5.1 正面信号
表格：序号 / 信号 / 详细说明。每条信号都要有具体依据（数字、事件、人名），不写空泛的正面评价。

### 5.2 风险信号
表格：序号 / 风险 / 详细说明。必须包含：阶段性风险（是否极早期、无产品/收入）、概念/技术路线风险、关联方/治理风险（如果有）、竞争格局风险（大厂/头部玩家的先发优势）、商业化不确定性。风险信号的数量和正面信号大致相当——报告的可信度取决于这种平衡。

### 5.3 核心待验证问题
给出一份为"下一次管理层会谈"准备的问题清单（5-8条），按优先级排列，每条都应该是访谈中能直接问出口、且答案会实质影响投资判断的问题（不要问"公司未来规划是什么"这种空泛问题，要问具体到"A和B两家关联公司的IP边界如何划分"这种）。

### 5.4 综合判断
- 投资评级（用机构常见说法，如"积极关注/建议深度尽调/暂不推进"等，跟用户确认措辞习惯）
- 正面判断段落（不是重复5.1，而是综合权衡后的结论）
- 保留判断段落（不是重复5.2，而是综合权衡后的结论）
- 关键催化剂：列2-3个会改变判断的未来事件节点
- 与已有业务/投资组合的关联（如果用户提供了这个上下文；没有就省略这条，不要编造）
- 免责声明：材料来源+编制基准日+"不构成投资建议"

## 附录：关键数据速查
### A.1 核心指标汇总表
把正文里出现过的所有关键量化指标（benchmark数字、财务数字、用户数字）汇总成一张表，方便读者不翻正文直接查数。字段：项目/来源 / 类别 / 关键指标 / 对比基准。

### A.2 信息源清单
列出本报告依据的全部材料（BP文件名+日期、补充材料文件名、通过网络搜索获取的信息及可追溯的来源说明）。这一节的作用是让读者知道哪些结论有据可查、哪些是分析师推断——务必如实列出，不要省略网络检索来源。
